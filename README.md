#Network Troubleshooting Lab

##Objective

Practice basic network troubleshooting techniques used in IT Help Desk roles.

---

###Tools Used

-Windows Command Prompt
-ipconfig
-ping
-tracert

---

## Step 1 – Check IP Configuration

Command used:

```
ipconfig
```

**Purpose:** Verify IP address assignment and default gateway configuration.

---

## Step 2 – Test Internet Connectivity

Command used:

```
ping google.com
```

**Purpose:** Confirm DNS resolution and internet connectivity.

---

## Step 3 – Trace Network Route

Command used:

```
tracert google.com
```

**Purpose:** Identify the path packets take across the network to reach a destination server.

---

## Results

The traceroute showed multiple network hops between the local system and Google's servers, confirming successful internet connectivity and proper routing through the ISP.

---

## Skills Demonstrated

- Network diagnostics
- Command-line troubleshooting
- Understanding packet routing
- Basic IT help desk troubleshooting workflow
